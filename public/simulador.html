Simulador de peticiones HTTP

Esta página imita la zona principal de Postman: eliges un método, escribes la dirección del recurso y envías la petición. Abajo tienes conceptos y un área para probar en vivo contra la API de este proyecto (/api/personas).

Glosario rápido

HTTPProtocolo de comunicación entre cliente (navegador, Postman, esta página) y servidor.
PeticiónMensaje que envía el cliente: método + URI + cabeceras y a veces cuerpo.
RespuestaMensaje que devuelve el servidor: código de estado (número), cabeceras y cuerpo (HTML, JSON…).
MétodoVerbo que indica la intención: GET leer, POST crear, DELETE borrar, etc.
EndpointRuta concreta en el servidor, ej. /api/personas. Suele usarse como sinónimo de “URL del recurso”.
CabecerasMetadatos de la petición o respuesta. Ej.: Content-Type: application/json dice que el cuerpo es JSON.
Cuerpo (body)Datos que viajan en la petición (típico en POST/PUT). En JSON va entre llaves.
Código de estadoNúmero de tres dígitos: 2xx éxito, 4xx error del cliente, 5xx error del servidor.

URI vs URL: ¿qué va en la barra de dirección?

URI (Identificador de Recurso)
“Nombre” o ruta del recurso. Ejemplo: /api/personas o /api/personas/673abc.... No dice solo cómo localizarlo en la red; a veces incluye solo la parte del camino.
URL (Localizador Uniforme)
Es una URI que además indica dónde está el servidor en Internet: incluye esquema (http/https), host (localhost:3000) y normalmente la ruta. Ejemplo completo: http://localhost:3000/api/personas.

En Postman y aquí escribes normalmente la URL completa. Si pones solo la ruta (/api/personas), esta página la completa con el mismo origen (window.location.origin) para que funcione contra tu servidor local.

Métodos HTTP más usados

GETObtener datos (listar o leer). No debe cambiar el servidor. Sin cuerpo habitualmente.
POSTCrear un recurso o enviar datos. Suele llevar body (ej. JSON).
PUTReemplazar por completo un recurso existente (REST).
PATCHActualizar solo parte de un recurso.
DELETEEliminar un recurso. A veces el id va en la URL.

Este proyecto expone GET, POST y DELETE en /api/personas.

¿Dónde va cada cosa (como en Postman)?
  • Método: selector a la izquierda (GET, POST…).
  • URL: campo grande central — ahí va la dirección completa o la ruta del endpoint.
  • Body: pestaña “Body” en Postman; aquí el área de texto JSON solo aplica a métodos que envían datos.
  • Headers: esta demo añade automáticamente Content-Type: application/json cuando envías JSON en POST.
  • Send: envía la petición y debajo verás el código de estado y el cuerpo de respuesta.
