Cierre: evaluar la solución y crear entregables

Bloom · Evaluar y crear: valoras la solución completa (seguridad, desempeño, alineación con el proyecto multimedia) y produces artefactos finales (informe, vídeo, mejoras). Viene después de métricas (Fase 4); el marco global está en el mapa.

Fase 1 · Mapa Fase 2 · Simulador Fase 3 · Laboratorio Fase 4 · Métricas

Qué cubre esta fase

  • Evaluar: contrastar resultados con criterios del proyecto (latencia aceptable, costo de consultas, riesgos), usando evidencias de la Fase 4.
  • Crear: informe técnico integrado, narrativa de decisiones, y piezas audiovisuales o extensiones acordadas en la rúbrica.
  • Seguridad y validación funcional: demostrar controles sobre el API y escenarios de prueba reproducibles.

Informe técnico (formato sugerido)

  1. Resumen y objetivos del proyecto multimedia.
  2. Decisiones de arquitectura de datos sustentadas en métricas (RA1) — enlaza tablas y gráficos de métricas.
  3. Modelo documental final y proyección de volumen / planes de consulta (RA2).
  4. API REST: rutas, autenticación o controles usados, riesgos mitigados (RA3).
  5. Vídeo colaborativo (enlace): recorrido funcional + comentario sobre seguridad y desempeño observado.
  6. Conclusiones y líneas de mejora.
Checklist rápido: ¿El vídeo muestra el mismo API que documentaste? ¿Las métricas citadas corresponden a capturas o exportaciones recientes? ¿El equipo explicita quién validó qué requisito?
